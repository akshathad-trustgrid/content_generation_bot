Webhook Payload Schema Guide

Your webhook receives a POST request with a JSON payload when you publish or save a draft. Here is the JSON schema:

{
  "event": "article_published | article_draft_saved",
  "platform": "linkedin | synq-blog",
  "timestamp": "2026-06-02T10:36:32.123Z",
  "socialText": "Article Title\n\nArticle content...",
  "article": {
    "id": "art_1780051364221",
    "title": "Article Title",
    "content": "Full markdown content...",
    "status": "published | draft_saved",
    "keywords": ["decentralized social", ...],
    "createdAt": "...",
    "updatedAt": "..."
  }
}
